# 墨流小说生成器 - 完整产品设计与实现文档

## 一、产品定位

**一句话描述**：用户只需输入一个主题，AI 自动规划大纲、逐章生成完整小说，支持多作品管理、后台持续生成、随时阅读。

**核心原则**：
1. **零门槛**：打开就能用，不需要理解任何技术概念
2. **不丢数据**：任何操作都不会丢失已生成的内容
3. **后台运行**：生成小说是后台任务，用户可以随时去看别的
4. **符合直觉**：操作逻辑跟现实世界的"写书-书架-读书"一致

---

## 四、数据持久化策略

### 4.1 存什么、存在哪

| 数据 | 存储位置 | 什么时候存 | 刷新后 |
|------|---------|-----------|--------|
| 小说配置（主题、章节数等） | localStorage | 每次修改 | ✅ 保留 |
| API 配置（地址、Key等） | localStorage | 每次修改 | ✅ 保留 |
| 所有历史作品 | localStorage | 每章完成时 | ✅ 保留 |
| 当前阅读位置 | localStorage | 切换章节时 | ✅ 保留 |
| 正在生成的任务 | localStorage | 每章完成时 | ✅ 恢复继续生成 |
| UI 状态（折叠、侧边栏等） | 内存 | - | ❌ 重置为默认 |

### 4.2 刷新页面后的恢复逻辑

```
应用启动
  │
  ├── 有未完成的生成任务？
  │     ├── 是 → 恢复生成引擎，显示书架页（任务在后台继续）
  │     └── 否 →
  │           ├── 有历史作品？
  │           │     ├── 是 → 显示书架页
  │           │     └── 否 → 显示工作台页
  │           └── 
  │
  └── 恢复上次的阅读位置
        └── 如果用户打开某作品的阅读器，跳到上次的章节
```

### 4.3 localStorage 安全策略

```javascript
// 每次写入前检查大小
function safeSetItem(key, value) {
  try {
    localStorage.setItem(key, value);
  } catch (e) {
    if (e.name === 'QuotaExceededError') {
      // 存储满了，提示用户删除旧的历史记录
      showToast('存储空间不足，请删除一些历史记录');
    }
  }
}
```

## 五、生成引擎设计

### 5.1 核心流程

```
开始生成
  │
  ├── 1. 创建历史记录（状态：生成中）
  │
  ├── 2. 计算下一章编号
  │     └── 从已完成章节中找最大 ID + 1
  │
  ├── 3. 生成大纲（调用 API）
  │     ├── 输入：主题 + 前文摘要 + "请规划这一章"
  │     └── 输出：标题 + 大纲
  │
  ├── 4. 生成正文（调用 API）
  │     ├── 输入：主题 + 大纲 + 前文结尾 + 系统提示词
  │     └── 输出：完整章节内容
  │
  ├── 5. 解析内容
  │     ├── 提取标题
  │     ├── 提取正文
  │     └── 统计字数
  │
  ├── 6. 保存到历史记录
  │
  ├── 7. 还有下一章？
  │     ├── 是 → 回到第2步
  │     └── 否 → 标记历史记录为"已完成"
  │
  └── 结束
```

### 5.2 生成引擎的运行规则

**关键原则：生成引擎是一个独立的后台任务，不依赖任何 UI 组件。**

```javascript
// 伪代码 - 生成引擎
class GenerationEngine {
  running = false;
  
  async start(recordId, config) {
    if (this.running) return; // 防止重复启动
    this.running = true;
    
    try {
      let chapterNum = this.getNextChapterNum(recordId);
      
      while (this.running) {
        // 检查是否需要停止
        if (shouldStop) break;
        if (达到章节数上限) break;
        
        // 生成一章
        await this.generateChapter(chapterNum, config);
        chapterNum++;
      }
    } finally {
      this.running = false;
      // 更新历史记录状态
    }
  }
  
  stop() {
    this.running = false;
  }
  
  async generateChapter(num, config) {
    // 大纲 → 正文 → 保存
  }
}
```

**生成引擎不受以下操作影响**：
- 切换页面（书架 → 阅读 → 书架）
- 关闭侧边栏
- 刷新浏览器（通过 localStorage 恢复）

**生成引擎在以下情况停止**：
- 用户点击"停止"
- 所有章节生成完毕
- API 请求连续失败 3 次
- 用户关闭浏览器（下次打开自动恢复）

### 5.3 错误处理

```
API 请求失败
  │
  ├── 网络错误（无网络、超时）
  │     ├── 自动重试 3 次，每次间隔 2秒
  │     ├── 3次都失败 → 暂停生成，显示错误提示
  │     └── 用户点击"重试" → 从失败的章节重新开始
  │
  ├── API 返回错误（401/403/500等）
  │     ├── 401/403 → API Key 问题，停止生成，提示用户检查
  │     ├── 429（限流） → 等待 30秒后重试
  │     └── 500 → 自动重试 3 次
  │
  └── 内容解析失败
        ├── 尝试用备用方式解析
        └── 解析失败 → 保存原始内容，标记为"已完成"
```

## 六、完整交互流程图

### 6.1 新用户首次使用

```
打开应用
  → 看到工作台页（空状态）
  → 输入小说主题
  → 设置章节数量（默认10）
  → 设置每章字数（默认3000）
  → 展开 API 配置，填入 Key
  → 点击"开始创作"
  → 自动跳转到书架页
  → 看到新作品出现在列表顶部，带"生成中"动画
  → 可以点"查看"进入生成进度页
  → 或者直接关掉浏览器去做别的事
  → 下次打开，看到书架页，生成还在继续
```

### 6.2 老用户打开应用

```
打开应用
  → 检测到有历史作品
  → 显示书架页
  → 如果有未完成的生成任务，后台自动恢复生成
  → 用户点击某作品"阅读"
  → 进入阅读器，从上次阅读的位置继续
  → 点击"上一章"/"下一章"切换
  → 点击"书架"回到书架
```

### 6.3 生成过程中

```
用户在生成进度页
  → 看到第4章正在生成中
  → 点击"书架"离开
  → 生成继续在后台运行
  → 书架页显示"生成中...已完成 4/10 章"
  → 用户点击另一个作品"阅读"
  → 进入阅读器
  → 生成继续在后台运行
  → 用户关掉浏览器
  → 下次打开，自动恢复生成
```

### 6.4 重新生成某章

```
在生成进度页
  → 看到第3章质量不好
  → 点击"重新生成"
  → 第3章状态变为"重新生成中..."
  → 重新走"大纲→正文"流程
  → 完成后更新第3章内容
  → 自动继续第4章的生成
```

## 九、技术实现要点

### 9.1 状态管理设计

```
┌─────────────────────────────────────────────────┐
│                   Zustand Store                   │
├─────────────────────────────────────────────────┤
│                                                   │
│  【持久化状态】（存 localStorage）                  │
│  ├── novelConfig: 小说配置                         │
│  ├── apiConfig: API 配置                          │
│  ├── history: 所有历史作品                         │
│  ├── currentRecordId: 当前活跃记录                  │
│  └── readingProgress: 阅读位置                     │
│                                                   │
│  【会话状态】（仅在内存中）                          │
│  ├── view: 当前视图                                │
│  ├── isGenerating: 是否正在生成                    │
│  ├── shouldStop: 是否需要停止                      │
│  ├── error: 当前错误信息                           │
│  └── sidebarCollapsed: 侧边栏状态                  │
│                                                   │
└─────────────────────────────────────────────────┘
```

### 9.2 组件层级

```
App
├── GenerationRunner (无 UI，始终挂载)
├── Header (导航栏)
├── Sidebar (条件显示)
└── Content (根据 view 切换)
    ├── ConfigPanel (工作台)
    ├── HistoryPanel (书架)
    ├── GeneratingPanel (生成进度)
    └── ReaderPanel (阅读器)
```

### 9.3 localStorage 键值设计

| Key | 内容 | 大小估算 |
|-----|------|---------|
| `ml-config` | API配置 + 小说配置 | < 1KB |
| `ml-history` | 所有历史作品（含正文） | 1KB - 2MB |
| `ml-active` | 当前活跃记录ID | < 50B |
| `ml-reading` | 阅读位置 | < 100B |

**注意**：`ml-history` 是最大的，每章约 5-10KB（3000字 × 2字节），10章约 50-100KB，100章约 0.5-1MB。localStorage 限制约 5MB，足够存 500+ 章。
