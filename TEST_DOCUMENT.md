# 墨流小说生成器 - 全面测试文档

## 一、配置页面测试 (ConfigPanel.tsx)

### 1.1 输入框边界条件

| 编号 | 测试场景 | 操作步骤 | 预期结果 | 实际风险 | 严重程度 |
|------|----------|----------|----------|----------|----------|
| C-01 | 章节数输入空值 | 清空章节数输入框，不输入任何内容 | 章节数为0（无限模式），字数为3000 | 用户可能不知道默认值是多少 | 中 |
| C-02 | 章节数输入非数字 | 在章节数输入框输入"abc" | `parseInt('abc')` 返回 `NaN`，`NaN \|\| 0` = 0，静默变成无限模式 | 用户不知道输入错误，无任何提示 | 高 |
| C-03 | 章节数输入负数 | 在章节数输入框输入"-5" | `parseInt('-5')` = -5，直接传给后端 | 后端无校验，-5章会导致生成逻辑异常 | 高 |
| C-04 | 章节数输入小数 | 在章节数输入框输入"5.7" | `parseInt('5.7')` = 5，静默截断 | 用户可能期望报错或提示 | 中 |
| C-05 | 章节数输入超大数 | 在章节数输入框输入"999999999" | 直接传给后端，无上限校验 | 后端无上限，可能导致极长等待 | 低 |
| C-06 | 章节数输入0 | 输入"0" | 0 = 无限模式，后端 target = 500 | 用户可能不知道0意味着500章 | 中 |
| C-07 | 字数输入空值 | 清空字数输入框 | `parseInt('')` = NaN，`NaN \|\| 3000` = 3000 | 默认值3000未显示在输入框中 | 中 |
| C-08 | 字数输入非数字 | 在字数输入框输入"xyz" | 同 C-02，静默变成3000 | 无错误提示 | 高 |
| C-09 | 字数输入负数 | 在字数输入框输入"-100" | `parseInt('-100')` = -100，传给后端 | 后端负字数会导致 max_tokens 为负数 | 高 |
| C-10 | 字数输入0 | 输入"0" | `parseInt('0')` = 0，`0 \|\| 3000` = 3000（注意：0是 falsy） | 用户输入0会静默变成3000，不直觉 | 高 |
| C-11 | 字数输入超大数 | 输入"1000000" | 直接传给后端，max_tokens = 2000000 | API 调用会失败，浪费 token | 中 |
| C-12 | 章节数输入框带空格 | 输入" 5 " | `parseInt(' 5 ')` = 5，正常解析 | 无问题 | 低 |
| C-13 | 章节数输入框带特殊字符 | 输入"5章" | `parseInt('5章')` = 5，静默截断 | 用户可能不知道截断了 | 中 |
| C-14 | 字数输入框带逗号 | 输入"3,000" | `parseInt('3,000')` = 3，只解析到逗号前 | 用户输入习惯导致错误 | 高 |
| C-15 | 字数输入框带单位 | 输入"3000字" | `parseInt('3000字')` = 3000，正常解析 | 无问题 | 低 |
